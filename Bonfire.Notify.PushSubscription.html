    Bonfire.Notify.PushSubscription (Bonfire v1.0.2-social-alpha.37)

    View Source

    Schema for a browser push subscription endpoint (device-level).

    This represents the browser's push service registration — one per service-worker origin + browser + device. Keys (p256dh, auth) are tied to the subscription itself, not to any particular user.

    Users are linked via Bonfire.Notify.UserPushSubscription (a Needle mixin), allowing multiple users to share the same endpoint.

    Supports both standard browser format and Mastodon API format:

    • Browser: %{"endpoint" => "...", "keys" => %{"p256dh" => "...", "auth" => "..."}}
    • Mastodon: %{"subscription" => %{"endpoint" => "...", "keys" => %{...}}, "data" => %{"alerts" => %{...}, "policy" => "..."}}
